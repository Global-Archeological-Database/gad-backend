# Sprint — 2026-06-18

## Session 1: Database & UI Bug Fixes

### Fixed Issues

#### 1. Submit Artefact Map — Not Scrollable/Zoomable
- **File**: [`LocationPicker.tsx`](/Users/aahwaanithsinharoy/projects/gad-frontend/src/components/shared/LocationPicker.tsx)
- **Root Cause**: Mixed controlled (`center`/`zoom`) and uncontrolled (`defaultCenter`/`defaultZoom`) props on the `<Map>` component caused re-render conflicts. The `overflow-hidden` CSS intercepted touch/wheel events, preventing map gestures from working.
- **Fix**: Removed `defaultCenter`/`defaultZoom` (kept only controlled props), changed `overflow-hidden` → `overflow-clip`, added `touchAction: 'none'`, added `onWheel`/`onTouchMove` stopPropagation handlers, and explicitly set `scrollwheel={true}` and `draggable={true}`.

#### 2. Main Map — Missing Artefact Markers
- **File**: [`MapExplorer.tsx`](/Users/aahwaanithsinharoy/projects/gad-frontend/src/components/map/MapExplorer.tsx)
- **Root Cause**: Timing race condition — `onBoundsChanged` fired on initial map render before artifacts loaded from API. When artifacts arrived asynchronously, `mapBounds` was already set, and viewport culling filtered artifacts outside the initial viewport. Also, `onBoundsChanged` handler had incorrect parameter type.
- **Fix**: Added `hasInitialData` state to defer viewport culling until after first successful data load. Changed `defaultCenter`/`defaultZoom` to controlled `center`/`zoom`. Fixed `onBoundsChanged` handler signature to receive map instance directly.

#### 3. UI Contrast — Light Mode Sharpness
- **File**: [`Header.tsx`](/Users/aahwaanithsinharoy/projects/gad-frontend/src/components/layout/Header.tsx)
- **Root Cause**: "Submit Artifact" button, user avatar, and auth dropdown used full-opacity `bg-primary` in light mode, creating stark contrast against the warm parchment background (#FDF8F0).
- **Fix**: Applied Tailwind opacity modifiers (`bg-primary/85`, `hover:bg-primary/70`) in light mode, with `dark:bg-primary` overrides to preserve full opacity in dark mode. Applied to: Submit button, desktop dropdown trigger, mobile sheet avatar, and mobile "Create Account" button.

#### 4 & 5. AI Analyse & Compare Features — Not Working

## Fix 6 (Emergency): Map Crash — "This page couldn't load" + Missing Markers

**Time:** ~2026-06-18 05:16 CEST

### Root Cause
The previous fix for `MapExplorer.tsx`'s `handleBoundsChanged` changed the callback signature from destructuring `{ map }` from the event object to receiving the parameter as a `google.maps.Map` instance. However, the `@vis.gl/react-google-maps` library's `Map` component `onBoundsChanged` callback passes a `MapEvent` object `{ map, type }`, not a `google.maps.Map` directly. The parameter was treated as a map instance and `.getBounds()` was called on the `MapEvent` object, throwing `TypeError: instance.getBounds is not a function`. This runtime error caused the entire map component to crash.

### Fix
- Changed parameter from `(map?: google.maps.Map)` to `(event: { map?: google.maps.Map })`
- Changed instance resolution from `map || mapRef.current` to `event?.map ?? mapRef.current`
- Now correctly destructures the `.map` property from the event object

### Commit
- [`64ba038`] fix: correct onBoundsChanged handler signature - library passes MapEvent object not map instance (frontend)

## Session 7: LocationPicker Map Fixes — Scroll/Zoom, Theme Selector, Coordinate Sync

**Commit:** [`ca73ddd`](https://github.com/Global-Archeological-Database/gad-frontend/commit/ca73ddd)
**Files changed:** `src/components/shared/LocationPicker.tsx` (+178, −12)

**Problem:** The LocationPicker map in the artifact submission form had three issues:
1. Map was not scrollable/zoomable — `touchAction: 'none'` and `onWheel`/`onTouchMove` `stopPropagation()` handlers blocked Google Maps' gesture system
2. No map type/theme selector — users couldn't switch to terrain, satellite, or dark modes
3. Bi-directional coordinate sync was broken — panning the map didn't update lat/lng inputs, and typing coordinates had a stale closure bug in the debounce handler

**Root Cause:**
- `touchAction: 'none'` on the container div prevented Google Maps canvas from processing touch gestures
- React synthetic event `stopPropagation()` on `onWheel`/`onTouchMove` intercepted events before the map canvas could process them
- No `onDragend` handler was attached to the Map component
- `useCallback` in `handleCoordInput` captured stale `latInput`/`lngInput` values in the debounced `setTimeout` closure

**Fix:**
1. **Scroll/Zoom:** Removed `touchAction: 'none'` and the `onWheel`/`onTouchMove` stopPropagation handlers. Added `contain: 'layout style paint'` for rendering isolation without blocking events.
2. **Theme Selector:** Added `MapTheme` type (`'streets' | 'terrain' | 'satellite' | 'dark'`), `MAP_THEME_OPTIONS` constant, `LIGHT_MAP_STYLE`/`DARK_MAP_STYLE` arrays, and a floating `Layers` button with dropdown menu (bottom-left of map). Uses `mapTypeId` for terrain/satellite, custom styles for streets/dark.
3. **Coordinate Sync:** Added refs (`latInputRef`, `lngInputRef`, `locationDataRef`) synced via `useEffect` — the debounced `setTimeout` now reads from refs instead of stale closure values. Added `onDragend` handler that reads `event.map.getCenter()` and updates both lat/lng inputs and calls `onChange`.

**Verification:** TypeScript compilation passes cleanly (`npx tsc --noEmit` exits with 0 errors). Deployed to `main` and pushed to GitHub.
