## Session: 2026-06-09 — Dual-Directory Consolidation

### Problem
The VS Code workspace (`~/gad-backend`) and Firebase deployment directory (`~/projects/gad-backend`) contained duplicate source code. Firebase resource files (firebase.json, .firebaserc, firestore.rules, firestore.indexes.json, storage.rules, apphosting.yaml, apphosting.emulator.yaml) only existed in the Firebase directory, creating ambiguity about the source of truth.

### Solution
Consolidated everything into `~/gad-backend` as the single source of truth. See [`ADR-007`](../decisions/ADR-007-resolve-dual-directory.md).

### Steps Executed
1. **Copied 7 Firebase resource files** from `~/projects/gad-backend/` to `~/gad-backend/`:
   - `firebase.json` — Firebase project config (Firestore, App Hosting, Storage, Emulators, Auth)
   - `.firebaserc` — Project alias (`default: global-archaeological-database`)
   - `firestore.rules` — Security rules with function-based access control
   - `firestore.indexes.json` — 6 composite indexes for artifacts collection
   - `storage.rules` — Cloud Storage security rules
   - `apphosting.yaml` — App Hosting config (nodejs20, 0-100 instances, 512MB, secrets)
   - `apphosting.emulator.yaml` — Local emulator env vars

2. **Updated `.gitignore`** — Added entries for Firebase agent artifacts (`.agents/`, `skills-lock.json`) and debug logs (`firebase-debug.*.log*`)

3. **Verified everything works**:
   - `npm test` — 41/41 tests passing
   - Server health endpoint — healthy response with all services connected
   - Source files confirmed byte-identical between both directories

4. **Archived Firebase directory** — Moved `~/projects/gad-backend` to `~/projects/gad-backend-archived`

5. **Documented decision** — Created [`ADR-007-resolve-dual-directory.md`](../decisions/ADR-007-resolve-dual-directory.md) with full analysis, decision, and consequences
