# Sprint — 2026-06-18

## Session 1: Database & UI Bug Fixes

### Fixed Issues

#### 1. Submit Artefact Map — Not Scrollable/Zoomable
- **File**: [`LocationPicker.tsx`](/Users/aahwaanithsinharoy/projects/gad-frontend/src/components/shared/LocationPicker.tsx)
- **Root Cause**: Mixed controlled (`center`/`zoom`) and uncontrolled (`defaultCenter`/`defaultZoom`) props on the `<Map>` component caused re-render conflicts. The `overflow-hidden` CSS intercepted touch/wheel events, preventing map gestures from working.
- **Fix**: Removed `defaultCenter`/`defaultZoom` (kept only controlled props), changed `overflow-hidden` → `overflow-clip`, added `touchAction: 'none'`, added `onWheel`/`onTouchMove` stopPropagation handlers, and explicitly set `scrollwheel={true}` and `draggable={true}`.

#### 2. Main Map — Missing Artefact Markers
- **File**: [`MapExplorer.tsx`](/Users/aahwaanithsinharoy/projects/gad-frontend/src/components/map/MapExplorer.tsx)
- **Root Cause**: Timing race condition — `onBoundsChanged` fired on initial map render before artifacts loaded from API. When artifacts arrived asynchronously, `mapBounds` was already set, and viewport culling filtered artifacts outside the initial viewport. Also, `onBoundsChanged` handler had incorrect parameter type.
- **Fix**: Added `hasInitialData` state to defer viewport culling until after first successful data load. Changed `defaultCenter`/`defaultZoom` to controlled `center`/`zoom`. Fixed `onBoundsChanged` handler signature to receive map instance directly.

#### 3. UI Contrast — Light Mode Sharpness
- **File**: [`Header.tsx`](/Users/aahwaanithsinharoy/projects/gad-frontend/src/components/layout/Header.tsx)
- **Root Cause**: "Submit Artifact" button, user avatar, and auth dropdown used full-opacity `bg-primary` in light mode, creating stark contrast against the warm parchment background (#FDF8F0).
- **Fix**: Applied Tailwind opacity modifiers (`bg-primary/85`, `hover:bg-primary/70`) in light mode, with `dark:bg-primary` overrides to preserve full opacity in dark mode. Applied to: Submit button, desktop dropdown trigger, mobile sheet avatar, and mobile "Create Account" button.

#### 4 & 5. AI Analyse & Compare Features — Not Working
- **Files**: [`ai.controller.js`](src/controllers/ai.controller.js), [`health.routes.js`](src/routes/health.routes.js), [`ArtifactAISection.tsx`](/Users/aahwaanithsinharoy/projects/gad-frontend/src/components/artifacts/ArtifactAISection.tsx), [`SimilarArtifactsSection.tsx`](/Users/aahwaanithsinharoy/projects/gad-frontend/src/components/artifacts/SimilarArtifactsSection.tsx), [`api.ts`](/Users/aahwaanithsinharoy/projects/gad-frontend/src/lib/api.ts)
- **Root Cause**: Multiple issues — no network diagnostics in error handling, no AI health check endpoint, `NEXT_PUBLIC_API_URL` might not be configured, auth token failures could silently break requests.
- **Fix**:
  - Backend: Added `detail` field to error responses (dev mode only) in AI controller
  - Backend: Added `GET /api/health/ai` endpoint to check Gemini model initialization
  - Frontend: Granular error messages for 401/Unauthorized, 429/Too Many Requests, network errors, and other failures in both AI sections
  - Frontend: Added console warning when `NEXT_PUBLIC_API_URL` is unset
  - Frontend: Wrapped `getIdToken()` in try/catch to prevent auth failures from breaking all API requests
  - Frontend: Wrapped `fetch()` in try/catch for descriptive NetworkError messages
